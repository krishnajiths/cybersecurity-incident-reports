# GitHub VS Code Supply Chain Attack

## Date
May 20, 2026

## Target
Github and its users

## Sector
Technology Sector

## Type of Attack
Supply Chain Attack

## Threat Actor
TeamPCP

## Attack Method
Attackers used a malicious VS Code extension disguised as legitimate software to compromise a GitHub employee device and gain unauthorized access to internal repositories.

## Impact
The malware gave attackers access to around 3,800 internal repositories.

## Data Compromised
Data comprised is internal repositories including Github source code as well as internal organization data.

## Response Taken
Github immediately removed the malicious VS Code extension, isolated the endpoint, then began incident response.

## Current Status
Ongoing

## Short Summary
TeamPCP a known hacker group for their creation of the Shai-Halud malware. Through a supply-chain attack the group was able to make a VS Code extension into a trojan. An unknowing GitHub employee downloaded the malware allowing TeamPCP to change repository permissions and gain unaothorized access to internal repositories. Although GitHub was able to quickly contain the malware, worry grows as countless developers wait to find out if their sensitive data had been compromised.

## Lesson Learned
Users should verify downloads and use security scans even when downloading software from trusted companies.

## Severity Rating
High
